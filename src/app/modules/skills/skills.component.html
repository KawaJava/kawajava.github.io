Umiejętności:

Języki oraz technologie, z którymi pracuję dłuższy czas i które dobrze znam:

  • Java – mój ulubiony język programowania. Skupiam się głównie na funkcjonalnościach wprowadzonych w wersjach 8+. Jego nauka jest dla mnie priorytetem, gdyż wiążę z nim swoją przyszłość zawodową.
  • Bazy danych – pracuję głównie z relacyjnymi bazami danych: MySQL, PostgreSQL, MS Access. Projektuję bazy składające się z kilkudziesięciu tabel. Potrafię:
    • tworzyć tabele i relacje między nimi,
    • używać JOIN-ów,
    • dodawać, aktualizować i usuwać rekordy,
    • tworzyć zapytania (kwerendy).
    Miałem również styczność z MS SQL i Oracle, językami (T-SQL, PL/SQL) oraz posiadam wiedzę o bazach NoSQL (MongoDB).
  • Git / GitHub – znam podstawy pracy w konsoli systemowej. Umiem pracować zarówno na pojedynczym, jak i na wielu branchach oraz tworzyć Pull Requesty. Swoje commity zamieszczam na głównym branchu za pomocą rebase’a (preferuję to rozwiązanie zamiast merge). Potrafię m.in.:
    • dodawać pliki,
    • commitować,
    • korzystać z stash, git clean, git reset i wielu innych,
    • pracować na wielu branchach,
    • tworzyć Pull Requesty.
    Praca z gitem nie stanowi dla mnie żadnego kłopotu.
  • Hibernate – potrafię mapować relacyjne bazy danych (relacje OneToOne, OneToMany, ManyToMany), używam JPQL, stosuję blokowanie danych, cache'owanie oraz batch'owanie. Korzystam zarówno z DAO, jak i Spring Data JPA.
  • Spring – mój ulubiony framework. Regularnie tworzę w nim nowe aplikacje. Znam wiele jego modułów, m.in.:
    • Spring Boot,
    • Spring Security,
    • Spring Web,
    • Spring Mail Sender,
    • Spring Scheduling,
    • Spring AOP,
    • Spring Data JPA,
    • Spring Validation,
    • Spring Liquibase.
  • Clean Code – piszę czysty i przejrzysty kod zgodnie z zasadami KISS, SOLID, DRY. Dbam o zrozumiałe nazewnictwo klas i metod (w języku angielskim).
  • Testowanie aplikacji – piszę testy jednostkowe i integracyjne. Uczę się techniki TDD, ponieważ uważam, że prowadzi do lepiej napisanego kodu.
  • Angular – ukończyłem 30-godzinny kurs Spring + Angular. Umiem napiać w nim proste strony internetowe, frontend dla aplikacji backendowych (zarówno prostych CRUDów jak i zdecydowanie bardziej złożonych), przeprowadzać walidację wprowadzonych danych w formularzach itp. Znam i często stosuję biblioteki typu: Angular Material, Angular FlexLayout. Obecnie uczę się bezpieczeństwa w Angularze, np. Angular Guard, obsługi tokenów JWT oraz dodawania animacji/wykresów.
  • TypeScript / JavaScript – preferuję TypeScript ze względu na jego podobieństwo do Javy. Używam głównie w Angularze, ale uczę się również czystego JavaScriptu.
  • HTML oraz CSS / SCSS – znam podstawy, które pozwalają mi tworzyć schludne i czytelne strony internetowe.
  • Maven – używam głównie do zarządzania zależnościami. Czasami korzystam także z Gradle.
  • REST API – znam jego główne cechy, poziomy zaawansowania i wykorzystuję w aplikacjach backendowych.
  • Projekt Lombok – korzystam z tej biblioteki w celu redukcji boilerplate kodu.

Pozostałe języki oraz technologie, z którymi mam lub miałem styczność:

  • JSON – używam przy tworzeniu aplikacji backendowych jako formatu wymiany danych.
  • C++ – język, w którym poznałem podstawy programowania (pętle, instrukcje warunkowe, tablice, listy, funkcje). Obecnie rzadko go używam.
  • Visual Basic – miałem z nim styczność na studiach, ale nie należy do moich ulubionych technologii. Poza projektami akademickimi już z niego nie korzystam.
